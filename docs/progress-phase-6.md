# Phase-6 进度说明

## 图片资源模型调整
- 槽位资源已拆分为原始路径、原始文件名、缓存图路径。
- 预览继续使用缓存图，导出命名改为读取原始文件名。

## 组合命名如何保证用原图名
- 导出组合命名优先取 `originalBaseName`。
- 若缺失原始信息，降级到缓存路径 baseName（避免命名失败）。

## 缓存 key 如何避免旧缓存误命中
- 标准预览缓存 key 由“绝对路径 + 文件大小 + 修改时间 + 生成后预览尺寸 + previewMaxEdge”组合后 hash。
- 旋转/镜像后的派生预览 key 还会加入原文件大小、修改时间、previewMaxEdge、旋转角度与镜像状态。
- 同一路径内容被替换时会生成新缓存文件，不再误复用。

- 已从 `AppController` 抽离到 `AutoLayoutPolicy`。
- `AppController` 仅负责流程编排和策略调用。

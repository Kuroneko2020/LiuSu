# 实现计划（更新到 Phase-6）

## 1. 本阶段目标
优先收敛“资源身份”和“缓存路径”混用问题，并完成一轮可用性收尾：
- 资源元信息拆分
- 组合命名严格使用原图名
- 缓存 key 提升可靠性
- 自动模式策略逻辑独立化
- 拖拽/构图反馈增强

## 2. 核心实现

### 2.1 图片资源模型
- 新增 `ImageResource`：
  - `originalPath`
  - `originalBaseName`
  - `cachePath`
- `SlotState` 改为持有 `ImageResource`，预览仍走 `cachePath`，命名走 `originalBaseName`。

### 2.2 导出命名修正
- 组合命名优先使用 `pageSlotOriginalBaseName()`。
- 若原始名缺失，降级到缓存路径 baseName（文档中明确）。
- 继续保留非法字符过滤、长度限制、重名避让。

### 2.3 缓存 key 修正
- `ImageService::normalizeAndCache()` key 包含：
  - 绝对路径
  - 文件大小
  - 最后修改时间
- 解决“同路径文件内容更新仍命中旧缓存”问题。

### 2.4 自动模式策略收敛
- 新增 `AutoLayoutPolicy`，从 `AppController` 抽离预设规则。
- 预设规则：
  - 均衡填充：按槽位与图片比例差异决定 Fit/Crop
  - 人像优先：横图旋转 + Crop
  - 证件照优先：Fit

### 2.5 交互反馈增强
- 槽位拖放时目标槽位高亮提示。
- 构图拖动激活状态增加边框提示。
- 语义保持：拖手柄交换，拖内容仅偏移构图。

## 3. 本阶段不做
- 自定义模板编辑器
- AI 智能识别/智能裁切
- 云同步、多图层、滤镜调色
- RAW/PSD/PDF
- HEIC/HEIF 完整稳定支持

## 4. 限制说明
- 当前容器缺 Qt6 SDK，无法本地完整编译与交互验证。
- 拖放细节在不同平台输入栈上仍可能有手感差异，后续继续打磨。

## 5. 下一阶段建议（Phase-7）
1. 新任务确认弹窗与未保存保护。
2. 自动模式策略参数化（阈值可配置）。
3. 缓存目录清理策略与容量上限。
4. 拖拽目标命中反馈与可视化进一步优化。

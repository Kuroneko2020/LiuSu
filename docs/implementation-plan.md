# 实现计划（更新到 Phase-5）

## 1. 目标
在不改架构的前提下，优先修正行为一致性与状态正确性：
- 新任务重置机制
- 自动模式预设真实生效
- 预览/缩略图/导出裁切一致
- 槽位状态重置与拖拽稳定性
- 缩略图缓存刷新可靠性

## 2. 本阶段实现

### 2.1 新任务上下文
- `ProjectState::startNewSession()` 统一重置页面并按模板创建新任务页。
- 首页手动模式与自动模式入口均切换到新任务上下文。

### 2.2 自动模式预设接入业务
- 均衡填充：按图片与槽位宽高比差异决定默认 Fit/Crop。
- 人像优先：横图默认旋转为竖向并使用 Crop。
- 证件照优先：默认 Fit（完整放入）以保留完整头像区域。

### 2.3 一致性收敛
- 导入统一标准化图像缓存（EXIF 已纠正）。
- 预览的 fill-crop 使用与导出相同的归一化偏移裁切模型（source clip 公式一致）。
- 缩略图使用导出渲染链路生成，避免三套逻辑分叉。

### 2.4 状态正确性
- 向槽位导入新图时重置 `rotation/mirrored/fillMode/cropOffset/zoom`。
- `cropOffset` 在切页、交换、缩略图、导出中都由 C++ 模型统一驱动。

### 2.5 缩略图缓存刷新
- 在缩略图 URL 上附加版本号 query，页面内容变化后强制刷新。

## 3. 本阶段不做
- AI 识别/智能构图
- HEIC/HEIF 完整稳定支持
- 自定义模板编辑器
- 云同步、多图层、滤镜调色、RAW/PSD/PDF

## 4. 已知限制
- 当前环境缺 Qt6 SDK，无法在本容器完成完整编译与交互验证。
- 拖放手感仍可能因平台输入栈存在细微差异，后续继续打磨。

## 5. 下一阶段建议（Phase-6）
1. 新任务确认弹窗与未保存保护流程。
2. 自动模式策略扩展（排序规则/分组规则）。
3. 拖拽反馈与可视化指示优化。
4. 构图偏移重置、锁定与更精细边界控制。

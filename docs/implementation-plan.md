# 实现计划（更新到 Phase-4）

## 1. 架构延续
- 保持 Qt 6 + C++ + QML 分层。
- 核心状态继续在 `ProjectState`。
- 图像标准化输入在 `ImageService`。
- 导出与页面缩略图渲染在 `ExportService`。
- 模板物理尺寸与槽位布局统一收敛到 `TemplateLayout`。

## 2. Phase-4 重点

### 2.1 已有偏差修复
1. 自动模式改为直出流程：选图 → 自动排版 → 直接导出，不进入编辑页。
2. 导出尺寸从“写死 6x4 inch”改为物理尺寸体系：默认 148×100mm，按 `mm/25.4*ppi` 计算像素。
3. 两张模板改为左右并排版式（2x1），四张/六张保持网格。
4. 预览与导出统一图像基础：导入时标准化（含 EXIF 纠正）后再被预览和导出共同使用。
5. 胶片栏改为真实页面缩略图渲染，而非首图代表。

### 2.2 新增交互闭环
- 槽位交换拖拽：通过专用手柄触发，支持交换/移动。
- 铺满裁切模式下内容拖动构图：偏移写入 C++ 模型，影响预览/缩略图/导出。
- `SlotState` 增加 `cropOffset`（并预留 `zoom` 字段）。

### 2.3 自动模式/设置联动
- 自动模式读取持久化导出相关设置。
- QSettings 持久化：自动模式预设、默认导出路径、记忆上次路径、默认格式、默认分辨率、默认裁切标记。

## 3. 本阶段明确不做
- 自定义模板编辑器
- HEIC/HEIF 完整稳定支持
- AI 智能裁切
- 云同步 / 多图层 / 滤镜调色
- RAW/PSD/PDF 支持

## 4. 环境限制
- 本地若缺 Qt6 SDK，无法完成完整编译验证。
- Qt Quick 拖拽细节可能依平台交互手感略有差异，当前实现优先稳定和可维护。

## 5. 下一阶段建议（Phase-5）
1. 拖拽交换体验打磨（视觉反馈、碰撞提示）。
2. 构图偏移加入边界可视化与重置按钮。
3. 缩略图渲染缓存与导出并发优化。
4. 自动模式增加更多策略（智能排序/批次规则）。

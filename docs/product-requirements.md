# Liusu（留素） - 产品需求文档（第一阶段）

## 1. 产品目标
构建 Liusu（留素）桌面端照片拼版工具（非修图软件），围绕固定模板（两张/四张/九张）快速完成 6 英寸相纸拼版，并保留高频手动微调能力。

## 2. 第一阶段范围

### 2.1 项目骨架
- 技术栈：Qt 6 + C++ + Qt Quick/QML。
- 分层：UI（QML）/ 状态管理（C++ Model）/ 图像处理服务层 / 导出服务层。
- 产物要求：可编译、可运行、具备后续扩展能力。

### 2.2 页面框架
- 首页（模板选择）
- 编辑页（手动布局）
- 导出页
- 设置页（基础占位）

### 2.3 首页交互（模板卡）
- 模板：两张、四张、九张。
- 默认仅显示模板预览与名称。
- 点击卡片展开，显示“手动布局”“自动布局”按钮。
- 同时仅允许一张卡片展开。
- 再次点击已展开卡片可收起。
- 点击空白区域收起。
- 按 Esc 收起。
- 不依赖 hover，兼容触控。

### 2.4 编辑页框架
- 中央预览区。
- 底部胶片/队列栏。
- 右上角：导出当前页、删除当前页。
- 单独“批量导入”入口。
- 进入编辑页自动创建空白页。
- 页面出现内容后自动标记为有效并进入队列。
- 编辑动作自动保存到项目状态（第一阶段先完成状态模型与调用流）。

### 2.5 模板槽位框架
- 三种模板均需具备槽位结构。
- 空槽位显示加号。
- 点击空槽位加号：一次导入一张（第一阶段使用占位导入逻辑）。
- 有图槽位可选中，仅选中槽位显示操作条。
- 操作条按钮：旋转 90°、镜像、填充模式切换（完整放入/铺满裁切）。

### 2.6 拖拽框架（第一阶段占位）
- 有图槽位角落显示拖拽手柄（用于未来槽位交换）。
- 图像内容拖动入口（用于未来铺满裁切偏移）。
- 两种拖动入口必须区分。

### 2.7 导出页框架
参数模型与 UI：
- 导出路径
- 导出格式（JPG/PNG）
- 文件名规则（组合命名/日期-序号）
- 导出分辨率（300/600/自定义 PPI）
- 裁切标记开关

### 2.8 设置页框架
当前设置页已暴露：
- 自动拼版默认 PPI、填充策略、方向策略、命名方案、裁切标记、原图导出开关
- 默认导出路径、记忆上次导出路径、默认导出格式
- 缓存目录、预览质量、缓存保留时间
- 纹理目录与刷新/打开入口
- 主题切换入口（当前仍是占位设置）

### 2.9 图片支持模块边界（接口先行）
当前导入过滤包含 JPG/JPEG、PNG、WebP、BMP、TIFF、HEIC/HEIF。
EXIF 方向会通过 `QImageReader::setAutoTransform(true)` 参与预览与导出尺寸判断。
HEIC/HEIF 仍依赖系统 Qt 图像插件；无插件时会导入失败，不应写成完整稳定支持。

## 3. 非目标（第一版不做）
- 自定义模板编辑器
- 修图（滤镜/调色/磨皮）
- 文字贴纸边框素材
- 多图层
- 账号系统
- 云同步
- AI 自动选图
- 人脸识别自动构图
- RAW/PSD/PDF 支持

## 4. 验收标准
1. 文档：`docs/product-requirements.md`、`docs/implementation-plan.md`。
2. Qt 6 + C++ + QML 项目初始化完成。
3. 四页面可跳转。
4. 首页模板卡交互满足要求。
5. 编辑页具备预览区、队列栏、右上角按钮、批量导入入口、模板槽位占位。
6. 导出页、设置页具备可扩展参数模型与基础 UI。
7. 结构清晰，支持第二阶段迭代。
